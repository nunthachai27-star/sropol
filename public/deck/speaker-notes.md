## Slide 6 — ระบบทำงานอย่างไร · เทคโนโลยีและสถาปัตยกรรม (3:00)

> *Technical depth slide. Pair the friendly diagram on the right with substantive technical specs on the left. Use this when the audience includes engineers, IT, or technical reviewers from the ministry.*

**Open with the architecture (right side):**
"ระบบ KK-LRMS ทำงานเป็น 3 ชั้น"
- "ชั้นบน: **27 โรงพยาบาล** ในเครือข่าย — ส่วนใหญ่ใช้ HOSxP, ระบบดึงข้อมูลผ่าน BMS Session API ทุก 30 วินาที"
- "ตรงกลาง: **KK-LRMS Cloud** — รวบรวม ประมวลผล และเก็บข้อมูล"
- "ชั้นล่าง: **แพทย์ พยาบาล ผู้บริหาร** เห็นข้อมูลผ่าน Server-Sent Events แบบเรียลไทม์"
- "ลูกศรทอง: แสดง data packets ไหลลงตามจริง — ทุก 30 วินาที จากชั้นบน, ทุกไม่กี่วินาทีจากชั้นกลาง"

**Walk the 4 technical pillars (left side):**

**01 · Integration (เชื่อมข้อมูลโรงพยาบาล)**
- "BMS Session API — pull-based HOSxP integration ผ่าน per-hospital tunnel + browser-only sync option (PHI ไม่ออกจาก network โรงพยาบาล)"
- "Webhook REST API — push-based สำหรับ รพ. ที่ไม่ใช้ HOSxP, มี API key + signature"
- "Auto-discharge — ตรวจจับและปิดเคสเมื่อ HOSxP ไม่ส่งซ้ำใน sync cycle ถัดไป"

**02 · Clinical Processing (การประมวลผลทางคลินิก)**
- "CPD Score คำนวณจาก 8 ปัจจัยทางคลินิก: gravida, ANC visits, GA, ส่วนสูง, น้ำหนัก, fundal height, U/S fetal weight, hematocrit"
- "Digital partograph ตามมาตรฐาน WHO — alert line + action line"
- "Risk threshold: CPD score ≥ 10 → แจ้งเตือนสูติแพทย์ที่โรงพยาบาลแม่ข่ายทันที"

**03 · Security + PDPA (ความปลอดภัย)**
- "Encryption at rest: AES-256-GCM สำหรับ name + CID fields"
- "CID hashing: SHA-256 — เลขบัตรประชาชนตัวจริงไม่เคยถูกเก็บเป็น plaintext ที่ไหนเลย"
- "Role-based access (admin · doctor · nurse · viewer) + audit log ทุกการเข้าถึง PHI"
- "DPA (Data Processing Agreement) ลงนามกับโรงพยาบาลแต่ละแห่ง"

**04 · Performance & Reliability (ประสิทธิภาพและความเสถียร)**
- "SQL query latency < 2 วินาที (SLA)"
- "SSE broadcast < 5 วินาที จาก sync เข้าจนผู้ใช้เห็น"
- "รองรับ 200 concurrent users พร้อมกัน"
- "463 unit + integration tests · 99.4% sync success rate · 87 วัน continuous uptime (production)"

**Point at the Stack pill row:**
"Production stack ทั้งหมดเปิดเผยได้ — Next.js 15, React 19, TypeScript 5 strict, PostgreSQL 16, Redis, NextAuth.js v5, SWR, Tailwind 4, Docker, Vitest"

**Close:**
"ไม่มี black box — ทุก layer ตรวจสอบได้ · ยินดีให้ทีม IT ของกระทรวงเข้ามา audit code, schema และ logs ได้ทันที"
